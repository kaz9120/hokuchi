# 記事設計リファレンス

執筆モードおよび壁打ちモードで、文章の中身だけでなく「読まれるための設計」を行うためのリファレンス。

---

## 構成テンプレート

記事のボリュームに応じて使い分ける。

### ショート（500〜800文字 / 目安30分で執筆）

書きたいことが1つだけあるとき。「とにかく出す」ための型。

```
[見出し画像]
フック（体験・エピソード・問い）  ← 3〜5行
本題（言いたいこと）              ← 10〜15行
結び（次のアクション or 一言）    ← 1〜3行
```

### スタンダード（1,500〜2,500文字 / 目安1〜2時間で執筆）

テーマを掘り下げたいとき。noteの標準的な記事。

```
[見出し画像]
フック                            ← 3〜5行
背景・前提の共有                  ← 5〜10行
[画像 or 区切り線]
本題① 具体的なエピソード・事実    ← 10〜15行
[画像: スクリーンショット、図解など]
本題② 考察・学び                 ← 10〜15行
結び                              ← 1〜3行
```

### ロング（3,000文字以上 / 目安3時間以上）

ノウハウ系・振り返り系など、情報量が多いとき。

```
[見出し画像]
フック                            ← 3〜5行
この記事で伝えること（1〜2文）
[画像 or 区切り線]
セクション1                       ← 各10〜20行
[画像]
セクション2
[画像]
セクション3
結び                              ← 1〜3行
```

---

## 画像挿入の原則

- 見出し画像は必ず設定する。見出し画像の有無でPV・スキ数に数十%の差が出る（note公式データ）
- 本文中の画像は「800文字に1枚」を目安に配置する。スクロールの息継ぎになる
- 画像の役割は3種類: ①内容の補足（スクリーンショット、図解）、②空気感の演出（写真）、③区切り（装飾ライン）
- スクリーンショットや図解は、直後の説明文とセットで配置する
- 見出し画像の推奨サイズは1280x670px。大きな造形や色面のあるシンプルなデザインが目に留まりやすい

## タイトル設計

タイトルは記事の「顔」。noteはSNS経由の流入が多いため、タイムラインで目に留まるかどうかが勝負。

### 原則

- 何が書いてあるか1秒で分かること。内容を隠さない。結論をタイトルに入れてよい
- 25文字以内に重要なキーワードを入れる。SNSや検索結果で省略されるため
- 固有名詞・数字・具体的なエピソードを含めると、内容がイメージしやすくなる
- 煽りや釣りタイトルは使わない。等身大のトーンを保つ

### 良いタイトルのパターン

| パターン | 例 |
|---------|-----|
| 具体的な体験＋テーマ | 「Slackの通知を全部切った。フルリモート半年の結論」 |
| 問い＋主観 | 「『仕事』と『好き』の境界線で。コミュニティについて考えたこと」 |
| 固有名詞＋意外性 | 「エンジニアが技術広報をやって気づいたこと」 |
| 数字＋スコープ | 「技術広報1年目にやった3つのこと」 |

### 避けるべきタイトル

| パターン | 例 | 理由 |
|---------|-----|------|
| まとめ記事風 | 「【完全保存版】エンジニアのためのSlack活用術10選」 | 個人noteの温度感と合わない |
| 煽り・釣り | 「知らないと損！SNSアカウント名の最強戦略」 | ハイカロリー。ブランド毀損のリスク |
| 内容が分からない | 「最近思ったこと」「ふと気づいたこと」 | クリックされない |

### タイトル提案の手順

1. 記事の内容から「一番伝えたいこと」を1文で抽出
2. その1文をもとに、上記パターンから3案を提案
3. Xでシェアされたときに目に留まるか、25文字以内に核があるかを確認

## タグ戦略

noteのタグは記事の発見性を左右する。適切なタグを選ぶことで、フォロワー以外の読者にリーチできる。

### 原則

1記事あたり3〜5個。多すぎると逆効果。3つの層でタグを構成する。

| 層 | 説明 | 例 |
|----|------|-----|
| 主題タグ | 記事のテーマそのもの | エンジニア、技術広報、Slack |
| ジャンルタグ | 記事の形式・カテゴリ | エッセイ、仕事、コミュニティ |
| トレンド/お題タグ | note公式のお題・コンテストに該当する場合 | 該当する公式お題タグ |

### タグ選定の手順

1. 記事の主題から主題タグを2〜3個選定
2. ジャンルタグを1〜2個追加
3. note公式の募集中のお題（https://note.com/contests）に該当するものがあれば追加
4. 選定したタグをnoteの検索欄で確認し、一定のボリュームがあるか確かめる
