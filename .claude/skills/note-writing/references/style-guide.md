# スタイルガイド詳細リファレンス

SKILL.md の「スタイルの核心」を深掘りした完全版。執筆前に一度目を通す。レビュー時の詳細チェックにも使う。

---

## A. 文のリズム（最重要）

### 短文優先

1文は60文字以内を基本とする。80文字を超えたら分割を検討する。「〜で、〜して、〜なので、〜です」のような多重接続は禁止。接続詞でつなげるくらいなら、句点で切る。

悪い例:
> 私がSlackの通知設定を見直そうと思ったきっかけは、フルリモートで働き始めてから半年が経った頃に、集中力の低下を感じるようになったことでした。

良い例:
> フルリモート半年。集中力が落ちてきた。原因はSlackの通知だった。

### 体言止めの活用

段落の締めや、リズムの転換点で体言止めを使う。ただし連続使用は3回まで。それ以上はくどい。

> 答えは単純で、通知のフィルタリング。優先度の高いものだけを受け取る設定にした。

### 文の長短を混ぜる

短文だけが続くと箇条書きのようになる。2〜3文の短文のあとに、少し長めの文を1つ入れてリズムを作る。

> 変数を1つに絞る。これが結論。
> ただ、ここに至るまでには半年くらいの試行錯誤がありました。

## B. ビジュアル・デザイン

### ひらがな推奨

漢字が続くと圧迫感が出る。迷ったらひらがなにする。

| 漢字 | ひらがな |
|------|----------|
| 事 | こと |
| 時 | とき |
| 僕 | ぼく |
| 今 | いま |
| 出来る | できる |
| 更に | さらに |
| 是非 | ぜひ |
| 殆ど | ほとんど |
| 様々 | さまざま |
| 沢山 | たくさん |

### 余白のリズム

- 1段落は80〜120文字
- 4〜5行続いたら必ず改行を入れる
- 「文字の壁」を作らない

## C. トーン＆マナー

### 低カロリー（Low Calorie）

以下のハイカロリーな修飾語は禁止。

> 超、圧倒的、神、衝撃、革命的、究極、最強、完璧、驚異的、劇的、抜群、格段、飛躍的、画期的、待望の、ついに、まさに、とにかく凄い

### 装飾禁止（アンチポエム）

以下のパターンは「雰囲気で書いている」兆候。見つけたら削る。

禁止パターン:
- 結びに情緒的な一文を足す（「そんな温かい場所を、これからも大切にしていきたいと思います」）
- 抽象的な比喩で締める（「それはまるで、〜のようでした」）
- 伝えたいことを繰り返し言い換える（1回言えば伝わる）
- 「〜なのかもしれません」で文を着地させ続ける（3回以上の連続使用は禁止）

判定基準: その文を削っても意味が変わらないなら、削る。

### ストレートな伝達

言いたいことを最短距離で書く。「背景→経緯→気づき→結論→余韻」ではなく「結論→背景→補足」。読者に「で、何が言いたいの？」と思わせたら負け。

悪い例:
> いろいろと試行錯誤を重ねる中で、少しずつ見えてきたことがあります。それは、通知というものが私たちの集中力に与える影響の大きさでした。

良い例:
> 通知は集中力を壊す。半年かけて実感した。

### 断定のバランス

事実や自分の経験は断定してよい。「通知を切ったら集中できた」に「かもしれません」は不要。主観的な意見や一般化のときだけ「〜だと思います」「〜かもしれません」を使う。

| 場面 | 表現 |
|------|------|
| 自分の体験・事実 | 断定する（「〜だった」「〜した」） |
| 自分の意見 | 「〜だと思う」 |
| 一般化・推測 | 「〜かもしれない」 |

### キザな言い回しを避ける

以下のような表現は使わない。

> 〜という確信に至った、〜という結論に達した、〜を断言できる、私の持論では、言うまでもなく、ご存知のとおり

### 共感を求める表現（禁止）

「観客にマイクを向ける」タイプの問いかけは禁止。

- 「〜ですよね？」「〜ではないでしょうか？」
- 「共感していただけると思いますが」
- 「皆さんも経験があると思いますが」
- 「一緒に〜しましょう」

ただし、問いかけ自体は禁止ではない。自分が面白いと思って口に出す類の問いかけはOK。共感を求めているのではなく、自己開示の一種。

## D. 構成・論理

### フック・ファースト

時系列順（背景→経緯→結果）で書かない。「一番面白かったこと」「自分が引っかかったエピソード」を冒頭に配置。そこから本題へ繋げる。

良い例:
> 「あ、焚き火の人ですよね！」
> 最近、イベントや勉強会でそう声をかけてもらえることが増えました。

悪い例:
> 私がXのアカウント運用を本格的に始めたのは2022年10月頃でした。当時から技術広報的な役割をしており…

### リニアな論理展開

複雑な入れ子構造（大見出し→中見出し）を避ける。読者の「なぜ？」を一つずつ消していく。

### 結びは短く終わる

結びの段落は3文以内。まとめ直し・振り返り・抱負は不要。最後の一文で「次のアクション」か「一番伝えたかったこと」を書いて終わる。

悪い例（ポエム化した結び）:
> 焚き火のように、ゆっくりと、でも確かに温まるような。そんなコミュニティを、これからも育てていきたいと思います。誰かの「ちょっと行ってみようかな」が生まれる場所であり続けたい。それが、私の小さな願いです。

良い例:
> 次回のイベントは3月。場所はまだ決まっていない。決まったらXで告知します。

## E. 法的・安全基準

### 最上級表現の禁止

「No.1」「世界初」「唯一」「最高」は、客観的証拠（公的データ等）がない限り使用しない（景表法/規約遵守）。

### コンテンツの安全性

- 特定の個人・技術・思想を攻撃しない
- 多様性を尊重し、あくまで「自分はこう思う」というスタンスを崩さない
- 山本は本名・所属を公開しているため、個人ブログでも企業ブランド（MOSH）への影響を考慮する

## F. 避けるべきAI的表現

以下の表現が出てきたら、人間らしく書き直す。

### 構成の癖

- 「まず」「次に」「最後に」の連発
- 「以下の通りです」「については」の多用
- 過度に整った構成（箇条書きの濫用）

### 表現の癖

- 「〜することが重要です」
- 「〜と言えるでしょう」
- 「〜において」「〜に関して」
- 「さまざまな」「多くの」の連発
- 「効果的に」「効率的に」

### 感情・装飾の癖

- 型にはまった感情表現（「とても嬉しく思います」）
- 過剰な丁寧さ
- 段落末に雰囲気のある一文を追加する
- 同じことを違う言葉で2回言う
- 抽象的な比喩で締める

## G. 文体統一ルール

| 項目 | ルール |
|------|--------|
| 一人称 | 「私」で統一（「僕」「自分」は使わない） |
| 句読点 | 「、」「。」を使用 |
| 数字 | 半角数字（例: 2022年、800人、3つ） |
| 固有名詞 | そのまま表記（例: Slack、LINE、X、MOSH） |

## I. 文法チェックポイント

- 一文の読点（、）は3個まで。それ以上なら文を切る
- 連続漢字は6字まで。超えたら「の」や助詞を入れる
- 二重否定を避ける（「〜しないわけではない」→「〜することもある」）
- ら抜き言葉を避ける（「見れる」→「見られる」）
- 同じ助詞の近接重複を避ける（「私は彼は好きだ」→「私は彼が好きだ」）
- 同じ接続詞を連続させない（「しかし〜。しかし〜」→「しかし〜。一方で〜」）
- 主述の対応を保つ
- 修飾語は被修飾語の近くに置く
